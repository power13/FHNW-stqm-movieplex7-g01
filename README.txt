In einem ersten Schritt haben wir den Code ins Git-Repo eingefügt. 
Anschliessend hat jeder von uns 2 Test-Klassen geschrieben.
Jede dieser Test-Klassen wurde in einem eigenen Feature Branch entwickelt. 
Anschliessend wurden diese ins zentrale Repo gepusht und mittels Pull Requests gemerged.
Die Pull Requests wurden dabei für das Code Review anderen Team-Mitgliedern zugewiesen, welche die Pull Requests (falls i.O.) dann gemerged haben.
Teilweise wurden nach dem Merge die Feature Branches entfernt, da diese nicht mehr benötigt werden.
Mit diesem Branch werden die entwickelten Features resp. Test-Klassen nun in einem Release-Branch abgebildet und danach in den master- (PR mit Assignment Dozent) sowie develop-branche gemerged.
